# Security Policy

## Supported Versions

This project is currently maintained on the `main` branch.
Security updates and fixes are applied to the latest version only.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to report
- Open a **GitHub Issue** and label it as `security`, **or**
- If the issue is sensitive, avoid public disclosure and contact the repository owner directly via GitHub.

Please include:
- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested mitigations

---

## Disclosure Policy

- Security issues will be reviewed and triaged as soon as possible.
- Fixes will be prioritized based on severity and impact.
- Once a fix is available, details may be shared publicly in a release note or issue comment.

---

## Scope

This policy applies to:
- Application code
- Configuration files
- Docker and CI configurations

Third-party dependencies should be reported to their respective maintainers when appropriate.

---

Thank you for helping keep this project secure.
